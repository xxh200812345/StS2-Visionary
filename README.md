# StS2-Visionary
为《杀戮尖塔2》（Slay the Spire 2）开发的视觉辅助插件，提供卡牌/奖励分支的选取帮助

**SpireLens 2** 是一款基于 Python 的《杀戮尖塔 2》视觉辅助插件。它通过 OCR 识别游戏实时画面，并结合预设的 CSV 数据库，在游戏之上提供一层“智慧透镜”，为玩家提供抓牌建议、奖励分支、商店评估及战斗提醒。

---

## 🚀 核心特性

* **全场景智能检测**：利用 SIFT 特征点匹配，精准识别关键决策时刻：
    * **开局奖励**：分析涅奥（Neow）的开局选项优劣。
    * **商店交易**：识别商品价格、卡牌强度及移除建议。
    * **战斗奖励**：提供普通/精英战斗后的三选一抓牌指导。
    * **BOSS 结算**：针对 BOSS 遗物及特殊奖励分支提供长远流派建议。
* **非侵入式 UI**：基于 PyQt6 的透明置顶图层，不修改游戏内存，避开封号风险，安全合规。
* **悬浮控制球**：局内提供可拖动的迷你按钮，支持一键开关提示层、强制刷新识别或切换数据包。
* **数据驱动引擎**：支持 CSV 自定义数据库，玩家可自由导入不同“职业大神”的评价体系。
* **高容错模糊搜索**：内置 Levenshtein 距离算法，解决 OCR 在复杂背景下（如闪烁特效）的识错问题。

## 🛠️ 技术架构

项目采用模块化设计，确保各组件解耦：

| 模块 | 功能描述 | 技术选型 |
| :--- | :--- | :--- |
| **`core.detector`** | 环境感知，判断当前是否处于商店或结算界面 | OpenCV (SIFT) |
| **`core.ocr_engine`** | 文字抓取，识别卡牌名称、价格等信息 | PaddleOCR |
| **`core.data_manager`** | 数据中枢，负责加载 CSV 并进行模糊匹配 | Pandas / Difflib |
| **`ui.floating_btn`** | 交互入口，局内悬浮控制按钮 | PyQt6 (Frameless) |
| **`ui.overlay`** | 信息渲染，在卡牌上方贴出提示文字 | PyQt6 (Transparent) |

## 📂 项目结构

```text
SpireLens2/
├── main.py                # 启动程序
├── config.yaml            # 配置文件（如识别频率、透明度）
├── core/                  # 核心逻辑
├── ui/                    # 界面逻辑
├── docs/                   # 技术文档库
├── anchors/               # 场景判断用的模板图片
├── data/                  # 存放提示语的 CSV 文件
└── resources/             # 按钮图标等静态资源
```

## 📝 待办清单 (Roadmap)

#### 第一阶段：底层框架与数据中枢（进行中）
- [x] 项目模块化架构设计
- [ ] **Data Manager**: 实现基于 `difflib` 的 CSV 模糊匹配引擎
- [ ] **Data Manager**: 支持多场景（Shop/Reward/Neow/Boss）的提示语解析
- [ ] **Config System**: 支持 `yaml` 配置文件（保存透明度、识别频率、快捷键等）

#### 第二阶段：视觉环境感知（核心难点）
- [ ] **Scene Detector**: 实现商店与普通选牌场景的 SIFT 模板匹配
- [ ] **Scene Detector**: 采集并增加 **“开局涅奥界面”** 与 **“BOSS 遗物选择界面”** 的锚点特征
- [ ] **OCR Engine**: 整合 PaddleOCR，实现针对《尖塔2》特定字体（如能量槽、价格数字）的微调识别
- [ ] **Coordinate Mapper**: 建立坐标转换系统，将 OCR 的相对坐标映射至屏幕绝对物理像素

#### 第三阶段：UI 交互与渲染
- [ ] **Floating Button**: 实现可拖动的局内悬浮球，支持右键菜单
- [ ] **Overlay**: 开发全屏透明穿透图层，解决鼠标点击穿透（TransparentForInput）问题
- [ ] **Overlay**: 设计“智能气泡框”，根据识别位置自动计算弹出方向（防止遮挡文字）

#### 第四阶段：整合与优化
- [ ] **Multi-threading**: 将 OCR 与特征匹配放入独立线程，防止 UI 界面卡死
- [ ] **Performance**: 引入场景哈希对比，若画面未变化则跳过 OCR，降低 CPU 占用
- [ ] **Deployment**: 使用 Nuitka 打包为单文件 EXE，内置 Paddle 预测库与模型
