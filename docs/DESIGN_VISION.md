# 📑 [技术方案] StS2-Visionary：全分辨率自适应场景感知与触发机制

---

## 1. 设计目标
本方案旨在解决《杀戮尖塔2》在不同分辨率（1080p、2K、4K）及不同显示模式下，如何准确、低耗地识别以下关键决策场景：
* **Neow (开局)**：涅奥的祝福四选一。
* **Shop (商店)**：商人的地毯/清理按钮。
* **Reward (战后奖励)**：三选一卡牌或金币/药水。
* **Boss Reward (BOSS 遗物)**：关底三选一遗物。

---

## 2. 核心感知算法：三级级联过滤 (Tiered Filtering)

为了平衡“高准确率”与“低 CPU 占用”，系统采用级联式检测流程：

### 第一级：帧差哈希过滤 (Frame Hash Filter)
* **原理**：计算当前截屏与上一帧的感知哈希（pHash）。
* **作用**：如果画面变动率低于 2%（如静止等待操作），则直接跳过后续计算，保护系统资源。

### 第二级：SIFT 特征点锚点匹配 (Feature Anchor Matching)
* **技术选型**：OpenCV SIFT 算法（具备旋转、缩放、亮度不变性）。
* **锚点定义标准**：
    * **唯一性**：选取该场景特有的、不随动态背景变化的 UI 元素（如按钮边框、特定图标）。
    * **稳定性**：避开带有粒子特效（火花、烟雾）的区域。
* **匹配阈值**：设定 $N \ge 15$ 个良好匹配点（Good Matches）为场景触发临界值。

### 第三级：语义逻辑验证 (Semantic Verification)
* **原理**：调用轻量化 OCR 识别特定坐标区域的标题文字。
* **示例**：若 SIFT 判定为 Reward 场景，则验证屏幕上方是否包含“选择一张卡牌”的字符串。



---

## 3. 分辨率中性化方案 (Resolution Neutrality)

由于玩家分辨率各异，直接使用像素比对会导致失效。本方案采用**“逻辑坐标系”**：

### 3.1 归一化投影
* **基准定义**：所有场景锚点（Anchors）和兴趣区域（ROI）均以 1080p 为参考基准，但在存储时转换为 **$[0, 1]$ 相对坐标**。
* **动态缩放系数**：
    $$Scale_{current} = \frac{Height_{current}}{1080}$$

### 3.2 动态 ROI 采样
* 系统不会对全屏进行 SIFT 计算。根据场景逻辑，定义动态采样区：
    * **Shop**: 采样屏幕右上角（清理按钮常驻区）。
    * **Boss Reward**: 采样屏幕中心 30% 区域（遗物生成区）。

---

## 4. 状态机转换逻辑 (Scene State Machine)

系统维护一个内部状态机，防止 OCR 重复触发：
1.  **Idle (空闲)**：低频（2Hz）进行 SIFT 扫描。
2.  **Triggered (已触发)**：检测到场景切换，执行 **一次性全量 OCR**。
3.  **Locked (锁定)**：场景未变动时，仅维持 Overlay 显示，不再执行 OCR。
4.  **Exit (退出)**：检测到锚点消失，销毁 Overlay 图层，回归 Idle。

---

## 5. 边界情况处理 (Edge Cases)

| 异常情况 | 应对策略 |
| :--- | :--- |
| **高动态背景 (如大雾)** | 使用高斯模糊预处理，滤除高频噪声特征点。 |
| **超宽屏 (21:9)** | 以高度为基准缩放，水平方向取中心裁切对齐。 |
| **窗口化拉伸** | 动态调用 `GetWindowRect` 实时更新像素缩放比率。 |
