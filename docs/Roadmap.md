# 🚀 StS2-Visionary：通用架构验证版 Roadmap

## 第一阶段：环境与基座搭建 (Infrastructure)
*目的：确保开发环境可复现，项目结构可打包。*

- [x] **虚拟环境管理**：
    - 建立 `venv` 或 `conda` 环境，配置 `requirements.txt`。
    - 引入核心依赖：`PyQt6` (UI), `opencv-python` (视觉), `paddleocr` (文字), `pandas` (数据), `pyyaml` (配置)。
- [x] **全局配置文件 (`config.yaml`)**：
    - 定义 `target_process`: 目标程序窗口标题（测试阶段可设为 "记事本"）。
    - 定义 `resource_path`: 各场景锚点图与 CSV 的相对路径。
    - 定义 `ocr_settings`: 识别语言、置信度阈值。
- [x] **日志与调试系统**：
    - 建立控制台日志输出，记录场景检测状态与 OCR 耗时。

## 第二阶段：全链路技术验证 (The "Hello World" Loop)
*目的：跑通从“看到画面”到“显示提示”的完整闭环，不限游戏。*

- [x] **窗口捕获模块**：
    - 实现基于窗口句柄（Win32 API）的截图，而非全屏截图，确保支持后台/覆盖模式。
- [x] **场景检测验证 (Detector MVP)**：
    - 准备一张简单的测试图片作为“锚点”。
    - 当识别到测试图片出现在指定程序窗口时，触发一次 OCR。
- [ ] **数据匹配验证 (Data Manager MVP)**：
    - 读取一个只有 3 行数据的 `test.csv`。
    - 验证模糊匹配是否能处理 OCR 的轻微误差。
- [ ] **基础覆盖层 (Overlay MVP)**：
    - 实现一个全屏置顶透明窗口。
    - **核心攻克**：将 OCR 识别出的文字坐标，成功转化为 Overlay 上的红色方框。

## 第三阶段：内容填充与专项适配 (Game-Specific Expansion)
*目的：回归《杀戮尖塔2》，处理复杂场景。*

- [ ] **StS2 资源采集**：
    - 采集 Neow、Shop、Reward、Boss 的高清锚点图。
    - 编写 StS2 专属提示语 CSV。
- [ ] **分辨率适配层 (Resolution Adapter)**：
    - 实现归一化坐标转换逻辑，确保从 1080p 到 4K 的坐标偏移修正。
- [ ] **UI 交互增强**：
    - 完成悬浮球、右键菜单、手动刷新功能。

## 第四阶段：性能优化与发布 (Final Polish)
- [ ] **多线程调度**：
    - 将 `Visual-OCR-Matching` 整个链条放入独立线程，UI 线程仅负责重绘。
- [ ] **增量更新机制**：
    - 引入哈希校验，画面不动时不重复执行匹配。
- [ ] **打包分发**：
    - 使用 Nuitka 打包，确保自带环境，解压即用。
